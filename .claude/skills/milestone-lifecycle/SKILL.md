# Milestone Lifecycle

Use this skill before scaffolding a milestone, when starting implementation from a `00 - milestone coordination.md` file, during execution of any milestone task file, or when preparing a milestone for UAT handoff. It is the router for the milestone-delivery execution model.

## Start Here

- Create or switch to the milestone branch before writing scaffold files.
- If dependency documents still have open questions, use [Decision Locking](../decision-locking/SKILL.md) first.
- Read [Overview](overview.md) for the purpose and principles.
- Read [Layout](layout.md) for milestone folders, branching, and spec file structure.
- Read [Template](milestone-template.md) for the reusable milestone spec.
- Read [Execution Flow](execution-flow.md) for ownership, task status, and agent execution.
- Read [Handoff](handoff.md) for dependencies, testing, blockers, and UAT readiness.

## Routing

- Need to close open decisions in dependency documents? Read [Decision Locking](../decision-locking/SKILL.md).
- Need the branch creation/switching rule? Read [layout.md](layout.md).
- Need the high-level rules? Read [overview.md](overview.md).
- Need folder and branch rules? Read [layout.md](layout.md).
- Need a milestone spec template? Read [milestone-template.md](milestone-template.md).
- Need status, ownership, and execution rules? Read [execution-flow.md](execution-flow.md).
- Need handoff, testing, and blocker rules? Read [handoff.md](handoff.md).

## Scope

Use this skill for any roadmap milestone, not only the Autonomous Market Intelligence roadmap.
